

# 校园交易平台的整体设计

课题名称：基于springboot+vue3完成一个校园二手交易平台。
课题简介：随着二手交易在校园场景中的普及，现有二手交易平台逐渐暴露出诸多适配性不足的问题：一是身份认证体系缺失，校外人员与校内人员身份难以区分，用户信息不透明易引发信任危机；二是监管机制不完善，平台自我监管无力，违规商品、假冒伪劣商品滋生，且缺乏贴合校园场景的监管规则；三是信息呈现杂乱无章，未针对学生需求进行精准分类，信息筛选耗时久，需求匹配效率低下；四是售后与纠纷处理机制缺失，交易纠纷难以快速解决，缺乏适配校园场景的纠纷调解途径。为解决上述问题，本课题拟打造一款校园专属二手交易系统，通过校园化的功能设计如校园数据隔离、校园身份认证等填补现有平台在校园场景中的应用空白

基本要求：

实现校园专属认证功能：支持 “学号 / 教职工号 + 校园邮箱（若有）” 注册模式，后台人工核验身份信息，个人主页展示认证状态与交易评价，解决现有平台身份不透明、校外人员混入、交易可信度低的问题。
实现校园化监管功能：商品发布时自动过滤违规品类，管理员后台支持批量审核与下架违规商品并生成统计数据，强制要求二手商品标注新旧程度、瑕疵说明且上传多张实物图，解决平台监管无力、商品质量无保障、违规风险高的问题。
实现精准信息匹配功能：按学生核心需求定制商品分类（考研资料、教材、宿舍用品、数码等），支持按校区、年级筛选信息，首页展示热门需求推荐，解决现有平台信息杂乱、筛选耗时、需求匹配效率低的问题。
实现轻量化纠纷处理与交易保障功能：留存交易订单完整聊天记录与商品描述作为纠纷凭证，设立 “校园调解” 入口并承诺 12 小时内响应，跑腿模块增加 “取件凭证上传” 进度确认功能，解决交易纠纷难解决、无校园化售后、虚假接单等问题。

### 4、数据库详细设计

#### 1. 学校表（school_info）

| 字段名      | 数据类型     | 说明         | 备注 / 业务意义                       |
| ----------- | ------------ | ------------ | ------------------------------------- |
| school_code | VARCHAR(16)  | 学校标准编码 | 主键，如 10001（与教育部编码对齐）    |
| school_name | VARCHAR(128) | 学校中文全称 | 唯一约束，如 “XX 大学”                |
| campus_code | VARCHAR(16)  | 校区编号     | 联合主键，如 1000101（主校区）        |
| campus_name | VARCHAR(128) | 校区名称     | 如 “XX 大学主校区”                    |
| status      | TINYINT      | 校区状态     | 0 - 禁用，1 - 启用（控制校区可用性）  |
| create_time | DATETIME     | 创建时间     | 默认 CURRENT_TIMESTAMP                |
| update_time | DATETIME     | 更新时间     | 自动更新，CURRENT_TIMESTAMP ON UPDATE |

```
CREATE TABLE `school_info` (
  `school_code` VARCHAR(16) NOT NULL COMMENT '学校标准编码（主键）',
  `school_name` VARCHAR(128) NOT NULL COMMENT '学校中文全称',
  `campus_code` VARCHAR(16) NOT NULL COMMENT '校区编号（联合主键）',
  `campus_name` VARCHAR(128) NOT NULL COMMENT '校区名称',
  `status` TINYINT DEFAULT 1 COMMENT '状态（0-禁用，1-启用）',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`school_code`, `campus_code`),
  UNIQUE KEY `uk_school_name` (`school_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学校&校区信息表';
```

#### 2. 商品分类表（item_category）

| 字段名        | 数据类型    | 说明      | 备注 / 业务意义                           |
| ------------- | ----------- | --------- | ----------------------------------------- |
| category_id   | INT         | 分类 ID   | 主键（自增）                              |
| category_name | VARCHAR(64) | 分类名称  | 唯一约束，如 “考研资料、教材”             |
| parent_id     | INT         | 父分类 ID | 0 为一级分类，支持多级分类（如数码→手机） |
| sort          | INT         | 排序权重  | 数字越小越靠前（首页分类展示顺序）        |
| status        | TINYINT     | 分类状态  | 0 - 禁用，1 - 启用                        |
| create_time   | DATETIME    | 创建时间  | 默认 CURRENT_TIMESTAMP                    |
| update_time   | DATETIME    | 更新时间  | 自动更新，CURRENT_TIMESTAMP ON UPDATE     |

```
CREATE TABLE `item_category` (
  `category_id` INT NOT NULL AUTO_INCREMENT COMMENT '分类ID（主键）',
  `category_name` VARCHAR(64) NOT NULL COMMENT '分类名称（如：考研资料、教材、宿舍用品）',
  `parent_id` INT DEFAULT 0 COMMENT '父分类ID（0为一级分类）',
  `sort` INT DEFAULT 0 COMMENT '排序权重（数字越小越靠前）',
  `status` TINYINT DEFAULT 1 COMMENT '状态（0-禁用，1-启用）',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`category_id`),
  UNIQUE KEY `uk_category_name` (`category_name`),
  KEY `idx_parent_id` (`parent_id`) COMMENT '父分类索引，支持多级分类查询'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品分类表';
```



#### 3. 用户信息表（user_info）

| 字段名          | 数据类型      | 说明            | 备注 / 业务意义                            |
| --------------- | ------------- | --------------- | ------------------------------------------ |
| user_id         | BIGINT        | 用户 ID         | 主键（自增 / 雪花 ID）                     |
| student_no      | INT           | 学号 / 教职工号 | 唯一约束（身份核验核心）                   |
| is_admin        | TINYINT       | 是否管理员      | 0-否，1-是                                 |
| image_url       | VARCHAR(255)  | 头像 URL        | 存储 OSS / 本地图片路径                    |
| user_name       | VARCHAR(25)   | 真实姓名        | 实名认证展示                               |
| phone           | VARCHAR(25)   | 手机号          | 预留短信通知                               |
| school_code     | VARCHAR(16)   | 学校编码        | 关联 school_info 表，多校数据隔离          |
| campus_code     | VARCHAR(16)   | 校区编码        | 关联 school_info 表，校区精准筛选          |
| auth_status     | TINYINT       | 认证状态        | 0 - 未认证，1 - 待审核，2 - 通过，3 - 拒绝 |
| credit_score    | INT           | 信用分          | 初始 100，交易评价动态调整                 |
| account_status  | TINYINT       | 账号状态        | 0 - 正常，1 - 封禁                         |
| runnable_status | TINYINT       | 跑腿认证状态    | 0 - 未认证，1 - 已认证                     |
| nick_name       | VARCHAR(25)   | 昵称            | 页面展示用（非真实姓名）                   |
| money           | DECIMAL(10,2) | 账户余额        | 模拟支付（无真实支付渠道）                 |
| follow_count    | INT           | 关注数（缓存）  | 默认 0，关注 / 取消关注时同步更新          |
| fan_count       | INT           | 粉丝数（缓存）  | 默认 0，他人关注 / 取消关注时同步更新      |
| create_time     | DATETIME      | 创建时间        | 默认 CURRENT_TIMESTAMP                     |
| update_time     | DATETIME      | 更新时间        | 自动更新，CURRENT_TIMESTAMP ON UPDATE      |

```
CREATE TABLE `user_info` (
  `user_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户主键ID',
  `student_no` INT DEFAULT NULL COMMENT '学号/教职工号（唯一）',
  `is_admin` TINYINT DEFAULT 0 COMMENT '账号状态：0-否，1-是',
  `image_url` VARCHAR(255) DEFAULT NULL COMMENT '用户头像URL',
  `user_name` VARCHAR(25) DEFAULT NULL COMMENT '用户真实姓名',
  `phone` VARCHAR(25) DEFAULT NULL COMMENT '手机号',
  `school_code` VARCHAR(16) NOT NULL COMMENT '学校编码（关联school表）',
  `campus_code` VARCHAR(16) NOT NULL COMMENT '校区编码（关联school表）',
  `auth_status` TINYINT DEFAULT 0 COMMENT '身份认证状态：0-未认证，1-待审核，2-通过，3-拒绝',
  `credit_score` INT DEFAULT 100 COMMENT '信用分（初始100分）',
  `account_status` TINYINT DEFAULT 0 COMMENT '账号状态：0-正常，1-封禁',
  `runnable_status` TINYINT DEFAULT 0 COMMENT '跑腿认证状态：0-未认证，1-已认证',
  `nick_name` VARCHAR(25) NOT NULL COMMENT '用户昵称（页面展示用）',
  `money` DECIMAL(10,2) DEFAULT 0.00 COMMENT '账户余额（模拟支付）',
  `follow_count` INT DEFAULT 0 COMMENT '关注数（冗余缓存：当前用户关注的人数）',
  `fan_count` INT DEFAULT 0 COMMENT '粉丝数（冗余缓存：关注当前用户的人数）',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '用户创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '用户信息更新时间',
  PRIMARY KEY (`user_id`),
  UNIQUE KEY `uk_student_no` (`student_no`),
  KEY `idx_school_campus` (`school_code`, `campus_code`),
  KEY `idx_auth_status` (`auth_status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户信息表（含关注数/粉丝数冗余字段）';
```



#### 4. 商品信息表（goods_info）

| 字段名         | 数据类型      | 说明            | 备注 / 业务意义                             |
| -------------- | ------------- | --------------- | ------------------------------------------- |
| product_id     | BIGINT        | 商品 ID         | 主键（自增 / 雪花 ID）                      |
| seller_id      | BIGINT        | 卖家 ID         | 关联 user_info 表，归属关系                 |
| category_id    | INT           | 分类 ID         | 关联 item_category 表，分类筛选核心         |
| school_code    | VARCHAR(16)   | 学校编码        | 多校数据隔离                                |
| campus_code    | VARCHAR(16)   | 校区编码        | 校区精准展示                                |
| trade_status   | TINYINT       | 交易状态        | 0 - 在售，1 - 已售出，2 - 下架              |
| description    | TEXT          | 商品详情        | 支持富文本（图文描述）                      |
| image_list     | TEXT          | 详情图 URL 数组 | JSON 格式存储（多张实物图）                 |
| image          | VARCHAR(255)  | 封面图 URL      | 首页 / 列表页展示                           |
| item_condition | TINYINT       | 商品成色        | 1-10 级（1 全新，10 破损严重）              |
| trade_type     | TINYINT       | 交易方式        | 0 - 仅线下，1 - 仅邮寄，2 - 皆可            |
| delivery_fee   | DECIMAL(5,2)  | 运费            | 仅邮寄模式生效                              |
| review_status  | TINYINT       | 审核状态        | 0 - 初审，1-AI 通过，2 - 人工通过，3 - 违规 |
| ai_valuation   | DECIMAL(10,2) | AI 定价建议     | 基于历史数据的智慧决策亮点                  |
| collect_count  | INT           | 收藏数          | 热度指标（首页热门推荐依据）                |
| price          | DECIMAL(10,2) | 商品价格        | 核心交易字段                                |
| create_time    | DATETIME      | 创建时间        | 默认 CURRENT_TIMESTAMP                      |
| update_time    | DATETIME      | 更新时间        | 自动更新，CURRENT_TIMESTAMP ON UPDATE       |

```
CREATE TABLE `goods_info` (
  `product_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '商品ID（主键，雪花ID可替换自增）',
  `seller_id` BIGINT NOT NULL COMMENT '卖家ID（关联user表）',
  `category_id` INT NOT NULL COMMENT '分类ID（关联item_category表）',
  `school_code` VARCHAR(16) NOT NULL COMMENT '学校编码',
  `campus_code` VARCHAR(16) NOT NULL COMMENT '校区编码',
  `trade_status` TINYINT DEFAULT 0 COMMENT '交易状态：0-在售，1-已售出，2-下架',
  `description` TEXT COMMENT '商品详情（富文本）',
  `image_list` TEXT COMMENT '详情图URL数组（JSON格式）',
  `image` VARCHAR(255) DEFAULT NULL COMMENT '封面图URL',
  `item_condition` TINYINT DEFAULT 5 COMMENT '商品成色（1-10级，5为中等）',
  `trade_type` TINYINT DEFAULT 0 COMMENT '交易方式：0-仅线下，1-仅邮寄，2-皆可',
  `delivery_fee` DECIMAL(5,2) DEFAULT 0.00 COMMENT '运费（邮寄时生效）',
  `review_status` TINYINT DEFAULT 0 COMMENT '审核状态：0-初审，1-AI通过，2-人工通过，3-违规',
  `ai_valuation` DECIMAL(10,2) DEFAULT 0.00 COMMENT 'AI定价建议',
  `collect_count` INT DEFAULT 0 COMMENT '收藏数',
  `price` DECIMAL(10,2) NOT NULL COMMENT '商品价格',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`product_id`),
  KEY `idx_seller_id` (`seller_id`) COMMENT '卖家ID索引，查询用户发布的商品',
  KEY `idx_category_id` (`category_id`) COMMENT '分类ID索引，主页按分类查商品',
  KEY `idx_school_campus_status` (`school_code`, `campus_code`, `trade_status`) COMMENT '学校+校区+状态，筛选本校在售商品',
  KEY `idx_review_status` (`review_status`) COMMENT '审核状态索引，筛选已审核商品'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品信息表';
```

```
CREATE TABLE `order_info` (
  `order_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '订单ID（主键）',
  `product_id` BIGINT NOT NULL COMMENT '商品ID（关联goods表）',
  `seller_id` BIGINT NOT NULL COMMENT '卖家ID',
  `buyer_id` BIGINT NOT NULL COMMENT '买家ID',
  `order_status` TINYINT DEFAULT 0 COMMENT '订单状态：0-待付款，1-待发货，2-待收货，3-已完成',
  `trade_time` DATETIME DEFAULT NULL COMMENT '交易完成时间',
  `amount` DECIMAL(10,2) NOT NULL COMMENT '订单金额',
  `school_code` VARCHAR(16) NOT NULL COMMENT '学校编码',
  `campus_code` VARCHAR(16) NOT NULL COMMENT '校区编码',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`order_id`),
  KEY `idx_product_id` (`product_id`) COMMENT '商品ID索引，关联商品信息',
  KEY `idx_buyer_seller` (`buyer_id`, `seller_id`) COMMENT '买家+卖家索引，查询用户的订单',
  KEY `idx_order_status` (`order_status`) COMMENT '订单状态索引，筛选待付款/待收货订单'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

## 5、短信发送功能

采用spug的短信发送，[Spug | 短信助手](https://sms.spug.cc/guide/发送短信)

Java调用示例

```
import java.io.*;
import java.net.*;

public class Demo {
    public static void main(String[] args) throws Exception {
        String url = "https://push.spug.cc/sms/A27Lsf3fs2bgEY";
        String json = "{\"name\":\"推送助手\",\"code\":\"153146\",\"to\":\"186xxxx9898\"}";

        HttpURLConnection conn = (HttpURLConnection) new URL(url).openConnection();
        conn.setRequestMethod("POST");
        conn.setRequestProperty("Content-Type", "application/json");
        conn.setDoOutput(true);

        try (OutputStream os = conn.getOutputStream()) {
            os.write(json.getBytes());
        }

        BufferedReader br = new BufferedReader(new InputStreamReader(conn.getInputStream()));
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
    }
}


```

### 请求参数[#](https://sms.spug.cc/guide/发送短信#请求参数)

| 参数名 | 类型   | 必填 | 说明                                                         |
| :----- | :----- | :--- | :----------------------------------------------------------- |
| name   | String | 否   | 根据模版内实际配置的变量名来传递，例如模版内配置了name变量，则需要传递name变量值 |
| code   | String | 否   | 根据模版内实际配置的变量名来传递，例如模版内配置了code变量，则需要传递code变量值 |
| to     | String | 是   | 手机号                                                       |

### 接口返回[#](https://sms.spug.cc/guide/发送短信#接口返回)

```json
{
  "code": 200,
  "msg": "请求成功",
  "request_id": "dwezjVRDoe5jgLkR"
}
```

### 接口返回说明[#](https://sms.spug.cc/guide/发送短信#接口返回说明)

| 参数       | 说明         | 描述                                                         |
| :--------- | :----------- | :----------------------------------------------------------- |
| code       | 状态码       | 200表示接口请求成功，不代表短信发送成功                      |
| msg        | 请求返回信息 | 请求成功或失败的原因，例如：请求成功、请求失败、请求参数错误等 |
| request_id | 请求ID       | 请求返回的ID，用于查询短信发送状态                           |

## 6、性能优化（数据库索引加固）

以下 SQL 脚本用于补全核心业务表的高频查询索引，大幅提升系统在大数据量下的统计、登录与列表查询性能。建议在系统大规模上线前执行。

```sql
-- 1. 用户信息表：登录接口性能优化
ALTER TABLE `user_info` ADD UNIQUE INDEX `uk_phone` (`phone`);

-- 2. 商品信息表：列表页排序与聚合优化
ALTER TABLE `goods_info` ADD INDEX `idx_create_time` (`create_time`);
ALTER TABLE `goods_info` ADD INDEX `idx_price` (`price`);

-- 3. 订单表：后台统计与个人订单查询优化
ALTER TABLE `order_info` ADD INDEX `idx_order_status` (`order_status`);
ALTER TABLE `order_info` ADD INDEX `idx_create_time` (`create_time`);
```
